# pharma_rd (application)

Python package for the **pharma_RD** multi-agent pipeline. This directory is the **uv** project root (nested under the `pharma_RD_workflow` monorepo).

## Prerequisites

- [uv](https://docs.astral.sh/uv/) (install: `curl -LsSf https://astral.sh/uv/install.sh | sh`, or `pip install uv`)

## Setup

From this directory (`pharma_rd/`):

```bash
uv sync --all-groups
```

Uses **Python 3.12** (see `.python-version`). The lockfile is `uv.lock`.

## Run

```bash
uv run pharma-rd --help
# or
uv run python -m pharma_rd --help
```

### On-demand pipeline run (FR1 / story 1.5)

Create a run, execute the full stub pipeline, then print a **summary JSON line** (after structured logs on stdout):

```bash
uv run pharma-rd run
```

- **Success:** exit code **0**. The **last** line of stdout is a JSON object: `run_id`, `poll_status` (placeholder `GET /runs/{run_id}` for a future HTTP API; use **`pharma-rd status <run_id>`** below for operator visibility without HTTP).
- **Failure:** exit code **non-zero**; error message on stderr.
- Uses **`PHARMA_RD_DB_PATH`** and **`PHARMA_RD_ARTIFACTS_ROOT`** like tests and `run_pipeline`.

### List runs & per-stage status (FR29 / FR31, story 1.6)

```bash
uv run pharma-rd runs
uv run pharma-rd runs --limit 50
uv run pharma-rd status <run_id>
```

- **`pharma-rd runs`:** prints **one JSON object** to stdout: `{"runs": [...]}`. Each run has **`run_id`**, **`status`**, **`created_at`**, **`updated_at`** (ISO 8601 strings from SQLite). Default **`--limit`** is **20** (newest runs first). If there are no runs, the list is empty and exit code is **0**.
- **`pharma-rd status RUN_ID`:** prints **one JSON object**: **`run`** (same fields as above) and **`stages`**, an array ordered **clinical â†’ competitor â†’ consumer â†’ synthesis â†’ delivery**. Each stage has **`stage_key`**, **`status`**, **`started_at`**, **`ended_at`**, **`error_summary`** (nullable). For **failed** stages, **`error_summary`** holds a short safe summary (exception type and message, capped); full stacks stay in structured logs, not in the DB.
- **Unknown `run_id`:** **`pharma-rd status`** exits **non-zero** and prints a clear message on stderr (no traceback in normal use).

Example **`runs`** output shape:

```json
{"runs":[{"run_id":"â€¦","status":"completed","created_at":"â€¦","updated_at":"â€¦"}]}
```

Example **`status`** output shape:

```json
{"run":{"run_id":"â€¦","status":"failed","created_at":"â€¦","updated_at":"â€¦"},"stages":[{"stage_key":"clinical","status":"completed","started_at":"â€¦","ended_at":"â€¦","error_summary":null},{"stage_key":"competitor","status":"failed","started_at":"â€¦","ended_at":"â€¦","error_summary":"RuntimeError: boom"}]}
```

### Retry a failed stage (FR30 / story 2.3)

When a run ends **`failed`** or **`partial_failed`** and a stage row is **`failed`**, you can **resume** from that stage using artifacts already written for upstream stages (no re-run of completed upstream work):

```bash
uv run pharma-rd retry-stage <run_id> <STAGE>
```

- **`STAGE`** is one of: **`clinical`**, **`competitor`**, **`consumer`**, **`synthesis`**, **`delivery`** (same order as **`pharma-rd status`**).
- **Preconditions:** the run is **not** **`completed`**; the target stage is **`failed`**; every **upstream** stage is **`completed`** and has **`output.json`** under the artifacts root. Otherwise the command exits **non-zero** with a short message on **stderr** (no traceback in normal use).
- **Success:** exit code **0**. The **last** line of stdout is JSON: **`run_id`**, **`poll_status`**, **`resumed_from`** (stage key). Structured logs include **`pipeline_resume`**, **`stage_retry`** per resumed stage, and **`stage_artifact_replaced`** when a stage output SHA-256 changes (NFR-R3).
- Use **`pharma-rd status <run_id>`** before/after to confirm stage rows and run status.

### Recurring schedule (FR2 / story 2.1)

Run a **long-lived in-process scheduler** (APScheduler **3.x**) that triggers the **same** full pipeline as **`pharma-rd run`** on a **cron** cadence:

```bash
uv run pharma-rd scheduler
```

- **Default cadence:** **`0 0 * * sun`** â€” minute/hour/day/month/day-of-week â€” **once per week**, **Sunday 00:00** in the configured timezone (PRD weekly default). **Note:** APSchedulerâ€™s `from_crontab` treats numeric **`0` in the DOW field as Monday**, not Sunday; use **`sun`** (or **`mon`**, etc.) for clarity.
- **Timezone:** `PHARMA_RD_SCHEDULER_TIMEZONE` (default **`UTC`**) â€” IANA name (e.g. `Europe/London`). Cron fields are interpreted in this zone.
- **Cron expression:** `PHARMA_RD_SCHEDULE_CRON` â€” standard **five-field** cron string. Invalid values cause **`pharma-rd scheduler`** to exit **non-zero** with a short message on **stderr** (no traceback in normal use).
- **Stdout:** Scheduled runs **do not** print the **`pharma-rd run`** summary JSON line on stdout; structured JSON logs from the pipeline still go to stdout (same as foreground runs).
- **Overlap:** If a tick fires while a run is still executing, the new tick is **skipped** and a **warning** is logged (`scheduler_skip`).
- **Config changes:** Restart the scheduler process after changing `.env` / environment (no hot reload in MVP). **`pharma-rd run`** and **`runs` / `status`** are unaffected by schedule settings.

## Configuration

- Copy `.env.example` to `.env` for local overrides. Use **non-secret** keys only; real secrets stay out of git (NFR-S1).
- Settings use the `PHARMA_RD_` prefix (see `pharma_rd/config.py`).
- **Database:** `PHARMA_RD_DB_PATH` (default `data/app.db`) â€” SQLite file for run/stage history; parent directory is created on open. Schema DDL and version live in `pharma_rd/persistence/db.py` (`PRAGMA user_version`). **v2+** adds `stage_artifacts` (path + SHA-256 per stage output). **v3+** adds nullable **`stages.error_summary`** for short failed-stage messages (NFR-R1).
- **Artifacts:** `PHARMA_RD_ARTIFACTS_ROOT` (default **`artifacts/`** under cwd) â€” JSON blobs per run/stage at `{root}/{run_id}/{stage_key}/output.json`; metadata in `stage_artifacts`. Default folder is gitignored.
- **Retention:** `PHARMA_RD_RETENTION_DAYS` (default **30**) â€” use `purge_runs_older_than()` from `pharma_rd.persistence` to delete runs older than this window (cascades stages and artifact rows).
- **Pipeline:** Ordered stages **clinical â†’ competitor â†’ consumer â†’ synthesis â†’ delivery**; contracts in `pharma_rd/pipeline/contracts.py`, orchestration in `pharma_rd/pipeline/runner.py` (`run_pipeline`).
- **Structured logs:** `PHARMA_RD_LOG_LEVEL` (default **INFO**) â€” `configure_pipeline_logging()` runs at the start of `run_pipeline` and emits **one JSON object per line** to **stdout** (implementation: `pharma_rd/logging_setup.py`, stdlib `logging` only). **Correlation:** `correlation_id` equals `run_id` for MVP. Each line includes at least: `timestamp` (ISO 8601 UTC), `level`, `message`, `run_id`, `correlation_id`, `stage`, `agent`, `event`, `outcome` (and run-level fields such as `completed_stage_count` / `run_status` where applicable). **Do not** log secrets, tokens, or PHI/PII (NFR-S4).
- **Schedule (Epic 2):** `PHARMA_RD_SCHEDULE_CRON` (default **`0 0 * * sun`**, weekly Sunday) and `PHARMA_RD_SCHEDULER_TIMEZONE` (default **`UTC`**). Implementation: `pharma_rd/scheduler.py` (APScheduler). See **Recurring schedule** above.
- **Outbound HTTP (Epic 2 / story 2.2):** Shared **`httpx`** settings for connector calls: `PHARMA_RD_HTTP_TIMEOUT_SECONDS` (default **30**), `PHARMA_RD_HTTP_MAX_RETRIES` (default **3** extra attempts after the first request), `PHARMA_RD_HTTP_RETRY_BACKOFF_SECONDS` (default **0.5**, exponential backoff). Optional **`PHARMA_RD_CONNECTOR_PROBE_URL`**: when set, each pipeline stage performs a **GET** through this client before stub agent logic (leave unset for offline/tests). On failure, structured logs may include **`integration_error_class`** (e.g. `timeout`, `transient_exhausted`); stage **`error_summary`** in the DB includes the classification for operators.
- **Clinical agent (Epic 3 / FR6):** **`PHARMA_RD_THERAPEUTIC_AREAS`** â€” comma-separated **therapeutic area** labels used to build a PubMed query: **OR** across **Title/Abstract** terms, **AND** **`Clinical Trial[Publication Type]`** (trial publications only). When **empty** (default), the clinical stage **still completes** with explicit **`data_gaps`** in the JSON artifact (NFR-I1) and does not call NCBI. When set, the agent calls **NCBI E-utilities** (`esearch` â†’ `efetch`) via the shared HTTP client; see **`pharma_rd/integrations/pubmed.py`**. Optional: **`PHARMA_RD_PUBMED_MAX_RESULTS`** (default **5**), **`PHARMA_RD_PUBMED_TOOL_EMAIL`** (recommended for responsible NCBI use), **`PHARMA_RD_PUBMED_EUTILS_BASE`** (override only for tests). **Practice/offline:** leave **`PHARMA_RD_THERAPEUTIC_AREAS`** unset to avoid outbound traffic; integration tests mock HTTP.
- **GPT clinical analysis (Epic 3 / story 3.3):** **`PHARMA_RD_OPENAI_API_KEY`** â€” **optional**. **Where to set it:** add to **`.env`** in this directory (`pharma_rd/`, copied from **`.env.example`**) or **export** in your shell before **`uv run pharma-rd run`** / **`pharma-rd scheduler`**; **`Settings`** loads `.env` like other **`PHARMA_RD_*`** variables. **When you need it:** only if you want the **OpenAI** pass after publication fetch and internal-research merge; if unset or whitespace-only, the stage **skips** the call and adds an explicit note to **`integration_notes`** (stage still **completes**). **`PHARMA_RD_OPENAI_MODEL`** (default **`gpt-4o`**) and **`PHARMA_RD_OPENAI_TIMEOUT_SECONDS`** (default **120**) tune the request. **`PHARMA_RD_INSIGHT_ORG_DISPLAY_NAME`** (default **`iNova`**) appears in analyst-style system prompts for white-labeling. **Failure policy:** auth, rate limits, or timeouts **do not** fail the clinical stageâ€”the artifact records **`integration_notes`** / **`data_gaps`** (NFR-R1, NFR-I2). **CI / tests:** no key required; **`call_clinical_gpt_analysis`** is mocked. **Never** commit real keys (NFR-S1).
- **Therapeutic area validation (Epic 8 / FR23):** At **`Settings`** load, **`PHARMA_RD_THERAPEUTIC_AREAS`** is normalized (trimmed labels, stored with commaâ€“space separators) and validated: **at most 32** labels, **at most 128** characters per label, **Unicode letters, digits, spaces, hyphen, underscore** only; **no** empty segments between commas. **Empty** (unset or whitespace-only) means **no TA scope** (NFR-I1). Invalid values raise at load time; **`pharma-rd`** exits **non-zero** and prints a validation error on **stderr**. Use **`Settings.therapeutic_area_labels()`** for the parsed list (do not split the raw string elsewhere).
- **Competitor watchlist validation (Epic 8 / FR24):** At **`Settings`** load, **`PHARMA_RD_COMPETITOR_WATCHLIST`** is normalized and validated like TA scope: **at most 32** labels, **at most 128** characters per label, **Unicode letters, digits, spaces, hyphen, underscore**, plus **`&`**, **`.`**, **`(`** **`)`** for corporate-style names; **no** empty segments between commas. **Empty** means **no watchlist** (NFR-I1; Competitor stage uses explicit **`data_gaps`**). Invalid values fail at load; **`pharma-rd`** exits **non-zero** on **stderr**. Use **`Settings.competitor_labels()`** for the parsed list.
- **Connection settings (Epic 8 / FR25):** Shared connector **timeouts / retries / backoff** are **`PHARMA_RD_HTTP_*`** (see **Outbound HTTP** above). **Base URLs** **`PHARMA_RD_PUBMED_EUTILS_BASE`**, **`PHARMA_RD_OPENFDA_DRUGSFDA_URL`**, optional **`PHARMA_RD_CONNECTOR_PROBE_URL`**, and optional **`PHARMA_RD_SLACK_WEBHOOK_URL`** must be valid **http(s)** URLs with a **host** at **`Settings`** load; **Slack** webhooks must use **`https://`**. Invalid URLs fail at load; **`pharma-rd`** exits **non-zero** on **stderr**. **`http_client.request_with_retries`** does not log bodies or secrets (NFR-S1).
- **Practice mode deployment (Epic 8 / FR26):** **`PHARMA_RD_DEPLOYMENT_PROFILE`** â€” **`practice`** (default), **`staging`**, or **`production`**. **Practice** is the demo-oriented build: **public/mock sources** are allowed and **enterprise SSO is not required** for MVP (**FR34** is roadmap). - **GPT synthesis (Epic 6 / story 6.5):** **`PHARMA_RD_SYNTHESIS_MODE`** â€” **`gpt`** (default) or **`deterministic`**. With **`gpt`** and **`PHARMA_RD_OPENAI_API_KEY`** set, the synthesis stage calls **OpenAI** (same **`PHARMA_RD_OPENAI_MODEL`** as other GPT steps, default **`gpt-4o`**) using JSON mode; the user prompt includes serialized **`ClinicalOutput`**, **`CompetitorOutput`**, and **`ConsumerOutput`** (including optional **`clinical_gpt_analysis`**, **`competitor_gpt_analysis`**, **`consumer_gpt_analysis`** when present). **`ranking_criteria_version`** is **`gpt_strategy_v1`**. **`aggregated_upstream_gaps`** (including **`[synthesis]`** truncation notes) and **`scan_summary_lines`** are still assembled **deterministically** from upstream artifacts for transparency. **`PHARMA_RD_OPENAI_SYNTHESIS_TIMEOUT_SECONDS`** (default **180**) bounds the synthesis request (NFR-P1). **Failure policy:** invalid JSON or **`SynthesisOutput`** validation failure triggers **one** bounded retry with a repair prompt; then the stage **fails** with an operator-oriented error (**no** silent fallback to deterministic). With **`gpt`** and **no** API key, synthesis **fails** unless you set **`PHARMA_RD_SYNTHESIS_MODE=deterministic`**. **`deterministic`** uses the legacy cross-domain ranking path (**`cross_domain_v1`**) without LLMâ€”use for **CI/offline** without a key. **`pytest`** defaults **`PHARMA_RD_SYNTHESIS_MODE=deterministic`** in **`tests/conftest.py`**. Large upstream lists are capped per domain (**40** items per list) before prompting; see **`pharma_rd/integrations/openai_synthesis.py`**.
- **Delivery / insight report (Epic 7 / FR18â€“FR19, FR22):** The delivery stage reads **`SynthesisOutput`** in memory from the runner, writes a **Markdown** report to **`{artifacts_root}/{run_id}/delivery/report.md`** and a **static HTML** twin to **`{artifacts_root}/{run_id}/delivery/report.html`** (UTF-8, atomic writes). **After** **`report.html`** exists on disk, the stage optionally **POST**s a **Slack Block Kit** notification when **`PHARMA_RD_SLACK_WEBHOOK_URL`** is set (see **Slack notifications** below); then it runs **distribution** (next bullet). It persists **`DeliveryOutput`** **`schema_version` 3** as **`{artifacts_root}/{run_id}/delivery/output.json`** with **`report_relative_path`**, **`report_format`** (**`markdown`**), **`report_byte_size`**, **`report_html_relative_path`**, **`report_html_byte_size`**, **`distribution_channel`** / **`distribution_status`** / **`distribution_detail`** (FR19), plus **`slack_notify_status`** / **`slack_notify_detail`** when Slack is used or skipped. **FR22 (human judgment):** disclaimer language is embedded in the **run summary** (executive callout), **each ranked opportunity**, the **Governance and disclaimer** section, and an HTML **`<footer>`**; wording lives in **`pharma_rd/agents/delivery.py`** for a single edit point. The report body also includes FR27/FR28 fields in the summary and ranked opportunities (no re-ranking).
- **GPT report narrative (Epic 7 / story 7.6):** **`PHARMA_RD_REPORT_RENDERER`** â€” **`template`** (default, CI) or **`gpt`**. The narrative GPT path runs only when **both** **`PHARMA_RD_REPORT_RENDERER=gpt`** **and** **`PHARMA_RD_OPENAI_API_KEY`** are set; an API key alone still uses the **template** renderer. With both set, delivery calls OpenAI once (JSON response with **`report_html`** body fragment + **`report_markdown`** + optional **`slack_executive_excerpt`**). HTML is sanitized with **`nh3`** before write (allowlist tags; **no** `<script>`; **`javascript:`** links stripped via explicit URL-scheme allowlist). A trusted shell adds **`<!DOCTYPE>`**, charset, base CSS, and the FR26 practice banner when applicable. If the model omits required FR22 phrases, the stage **prepends** the executive FR22 banner. **`PHARMA_RD_OPENAI_REPORT_DELIVERY_TIMEOUT_SECONDS`** (default **150**) bounds the call (NFR-P1). **`PHARMA_RD_REPORT_GPT_FALLBACK_ON_ERROR`** (default **true**): on API/parse failure, empty fields, or **thin** model output, delivery **falls back** to the string-template renderer and logs **`delivery_gpt_fallback`** (NFR-R1). **`gpt`** without an API key logs **`delivery_gpt_skipped_no_key`** and uses the template. Slack uses **`slack_executive_excerpt`** for the executive block when present (truncated). Implementation: **`pharma_rd/integrations/openai_report_delivery.py`**, **`report_html_sanitize.py`**.
- **Slack notifications (Epic 7 / FR19 extension):** **`PHARMA_RD_SLACK_WEBHOOK_URL`** â€” optional **incoming webhook** URL ([Slack incoming webhooks](https://api.slack.com/messaging/webhooks)). When **unset** or empty, **no** HTTP request is made; structured logs record **`slack_notify_skipped`** (INFO). When set, the pipeline sends **`blocks`** (Block Kit) plus a plain **`text`** fallback. The message includes **UTC run date** (calendar date at delivery execution), **pharma_RD** branding, **`signal_characterization`**, top ranked opportunities, therapeutic-area and competitor-watchlist scope from settings, an **FR22** disclaimer aligned with the report, and a **relative path** to **`report.html`** under the artifact root (MVP; optional future **`base_url`** for HTTPS links). **`PHARMA_RD_SLACK_WEBHOOK_TIMEOUT_SECONDS`** defaults to **10** seconds (separate from connector HTTP). Webhook POSTs do **not** follow HTTP redirects (**`follow_redirects=false`**). Webhook failures do **not** fail the delivery stage; logs use **`slack_notify_failed`** with actionable detail (**no** full URL or secrets). Implementation: **`pharma_rd/integrations/slack_insight_notification.py`**.
- **Report distribution (Epic 7 / FR19):** **`PHARMA_RD_DISTRIBUTION_CHANNEL`** â€” **`none`** (default, explicit skip in logs), **`file_drop`**, or **`smtp`**. **File drop (MVP):** set **`PHARMA_RD_DISTRIBUTION_DROP_DIR`** to a directory root; after **`report.md`** and **`report.html`** are written, the pipeline copies them to **`{drop_dir}/rd/{run_id}/`** and **`{drop_dir}/marketing/{run_id}/`** and writes **`{drop_dir}/{run_id}/manifest.json`** with absolute paths for audit (including HTML paths when present). **`smtp`** is reserved: distribution logs **`distribution_failed`** with **`error_type=smtp_not_implemented`** and **`distribution_status=failed`** (no credentials logged). Placeholder recipient envs for future SMTP: **`PHARMA_RD_RD_RECIPIENT_EMAIL`**, **`PHARMA_RD_MARKETING_RECIPIENT_EMAIL`**. **Product choice (fail-fast):** distribution failures do **not** abort the delivery stageâ€”the run still writes **`DeliveryOutput`** with **`distribution_status=failed`** and structured **`distribution_failed`** logs (NFR-R1).
- **For recipients (Epic 7 / FR20â€“FR21, NFR-P2, FR22):** **R&D** and **marketing** receive the **same** files for each **`run_id`** (separate folders for role audit only). **Markdown:** open **`report.md`** in any text editor or Markdown-aware tool (e.g. **VS Code**, **Cursor**, GitHub/GitLab preview). **Browser (no extension):** double-click **`report.html`** or open it via **File â†’ Open** in **Chrome**, **Edge**, or **Safari** (`file://` is fine for local copies). **Email:** attaching **`report.md`** yields readable plain text; **`report.html`** can be opened in a browser if the mail client saves attachments. No **pharma_RD**-specific desktop client is required. **FR22:** read the executive callout, per-opportunity lines, governance section, and HTML footer so **recommendations** are clearly **not approvals** and **pursuit remains human-owned**.
